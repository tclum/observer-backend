Part 1 — Set up Airtable

1
Create a new Base
Log in to Airtable → click Add a base → choose Start from scratch → name it Observer Data.
2
Rename the default table to "Observations"
Double-click the tab at the bottom that says Table 1 and rename it to exactly: Observations
3
Add these fields to the table
Delete any default fields Airtable creates, then add the following — all as Single line text type:
Field nameTypeContains
Entry IDSingle line textObservation number
TimestampSingle line textISO date/time
TypeSingle line textfull, quick, or missed
ObserverSingle line textObserver's login ID
LocationSingle line textLocation entered at session start
SessionSingle line textSession name
Person #Single line textPerson number within group
AgeSingle line textAge range
GenderSingle line textGender presentation
EthnicitySingle line textRace / ethnicity
DisabilitySingle line textVisible disability / aids
EmotionSingle line textEmotional state
MissedSingle line textYES or NO
4
Copy your Base ID
Look at the URL in your browser while viewing the base:
https://airtable.com/appXXXXXXXXXXXXXX/tblXXXX/...

The part starting with app is your Base ID. Copy and save it — you'll need it shortly.
5
Create a Personal Access Token
Go to airtable.com/create/tokens → click Create new token.

— Name it anything (e.g. Observer App)
— Under Scopes, add: data.records:write
— Under Access, select your Observer Data base
— Click Create token and copy it immediately (it won't show again)

Part 2 — Deploy the backend to Vercel

6
Create a new GitHub repository
Go to github.com/new → name it observer-backend → set to Private → click Create repository.
7
Upload the backend files
You received a folder called observer-backend with this structure:
observer-backend/
  api/
    sync.js ← the serverless function
  package.json
  vercel.json
  .gitignore
On your new GitHub repo page, click uploading an existing file → drag the entire observer-backend folder contents in → click Commit changes.
8
Import to Vercel
Go to vercel.com/new → click Import next to your observer-backend repo → leave all settings as default → click Deploy.

Wait ~30 seconds. Vercel will give you a URL like:
https://observer-backend-xxxx.vercel.app
9
Add your Airtable credentials as environment variables
In Vercel: go to your project → Settings → Environment Variables → add these three:
Variable nameValue
AIRTABLE_TOKENYour Personal Access Token (starts with pat...)
AIRTABLE_BASE_IDYour Base ID (starts with app...)
AIRTABLE_TABLE_NAMEObservations
After adding them, go to Deployments → click the three dots on the latest deployment → Redeploy so the variables take effect.
10
Copy your sync URL
Your sync endpoint is your Vercel URL with /api/sync appended:

https://observer-backend-xxxx.vercel.app/api/sync

This is the only URL your observers need. Keep it — you'll paste it into the app.

Part 3 — Connect the Observer app

11
Open observer_app.html and paste the sync URL
Open the app in your browser → sign in → on the New session screen, paste your Vercel sync URL into the Server URL field.

The dot next to "Sync server" will turn green and show your domain. You're connected.
12
Test it
Log a few observations → tap End → watch the sync banner. If it turns green, open your Airtable base — you should see rows appearing in the Observations table within a few seconds.
Sharing with your team — once deployed, simply share the observer_app.html file with each observer and tell them to paste the sync URL at the start of each session. No installation, no accounts required on their end.

Troubleshooting

Sync fails with "Server error"

Usually means the environment variables aren't set correctly in Vercel, or the deployment wasn't redeployed after adding them. Go to Vercel → Settings → Environment Variables, double-check all three values, then redeploy.

Sync fails with "Airtable returned an error"

Usually a field name mismatch. Make sure every field in your Airtable table is spelled exactly as listed in Step 3, including capitalisation and spaces.

The dot stays grey even after pasting the URL

Make sure the URL starts with https://. The dot turns green as soon as a valid URL is detected.

App works but no data appears in Airtable

Check that the table is named exactly Observations (capital O, no trailing space). Also confirm your Personal Access Token has data.records:write scope on the correct base.

Security note — the sync URL itself is not secret (it has no credentials in it), but anyone who has it can post data to your Airtable. For an internal team tool this is fine. If you ever want to restrict access, you can add a shared secret header — let your developer know.
